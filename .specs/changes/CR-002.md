# CR-002 — UI Quality Uplift v1 (5 screens)

## Summary

Visual quality uplift toàn bộ 5 screens shipped (Home, ProjectLanding, FeatureDetail, Search, Login) theo new charter [visual-language.md](../ui/visual-language.md). Establish design bar cao hơn cho catalog identity, hero panels, hover rewards, empty patterns. Không thay đổi FR/AC, chỉ visual + UX detail.

## Motivation

User feedback ngày 2026-04-25 sau khi M2 closed: catalog `/` "quá xấu", design hiện tại "quá đơn giản". Mặc dù Nexlab DS đã adopted (ADR-003), code chưa leverage được rich tokens (primary ramps, secondary chips, hero gradients, hover rewards). 5 screens shipped đang ở mức "barely functional Nexlab" thay vì "polished Nexlab".

Cụ thể problems quan sát được trên `/`:

- Mọi project row visually identical — name là tín hiệu duy nhất phân biệt.
- "(chưa có mô tả)" italic chiếm không gian khi đa số projects chưa có description → noise.
- Title `text-base` quá nhỏ, metadata plain text nhạt.
- Hover chỉ `bg-muted/50` — không reward interaction.

Tương tự 4 screens còn lại đều đang flat/monochrome.

Charter [visual-language.md](../ui/visual-language.md) lock baseline (commit `cf0cf1c`) → CR này thực thi charter qua 5 screens.

## Proposed change

5 screens × per-screen UI spec update + code refactor. Tuân charter §1-9. Per-screen visual structure refresh chi tiết trong từng `.specs/ui/<screen>.md` §Visual structure (Phase 1B commits).

### Order

User journey: Home → ProjectLanding → FeatureDetail → Search → Login (Login đơn giản nhất, làm cuối).

### Cadence

- Phase 1B: present design draft per screen → user review → commit spec update.
- Phase 2: code per screen → smoke + visual audit → commit.
- Phase 3: final sync (CHANGELOG cut + status flips).

## Decision

**Approved** 2026-04-25 — Option C (5-screen uplift).

**Amendment 2026-04-25** (post-reference review): bumping visual ambition to "Workspace style" (data-rich layout với hero stat chips, 2-col grid, progress bars, section dots, decorative marks, sticky activity rails, tabs). Charter v2 captures patterns. Skeleton-UI policy: build full reference UI, dummy/placeholder cho data chưa có backend (presence, activity log, tabs Activity/Members/Settings, follow button). Gate 1 home.md "full-width row" decision **revoked** — switch to 2-col card grid for richer per-project info density.
