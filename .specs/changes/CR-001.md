# CR-001 — Split monolithic `.env.local` into per-layer env files

<!-- template: 05-change-request-template.md@0.1 -->

## Summary

Tách `.env.local` (hiện ở repo root, chứa biến của cả infra + API + Web) thành **3 file độc lập** theo layer: `infra/docker/.env`, `apps/api/.env.local`, `apps/web/.env.local`. Mỗi layer self-contained, không leak biến sang layer khác.

## Motivation

- **Separation of concerns**: biến infra (POSTGRES*PORT, POSTGRES_USER, POSTGRES_PASSWORD, REDIS_PORT) không liên quan FE/BE runtime. Biến FE (`VITE*\*`) không liên quan API. Giữ chung làm giảm cohesion của mỗi layer.
- **Leak risk**: API secret (SESSION*SECRET) không nên visible trong thư mục FE dev; Vite scan `.env.local` sẽ expose bất cứ biến nào prefix `VITE*` — giảm rủi ro vô tình thêm secret với prefix sai.
- **Deploy parity**: production sẽ mỗi container đọc env riêng (compose infra, Dockerfile API, static FE build-time env). Layout dev nên phản ánh cùng mental model, tránh "works on dev, fails in prod".
- **Vite alignment**: Vite auto-load `.env.local` từ `apps/web/` — hiện đang workaround bằng cách để file ở root + chỉ khai báo `VITE_*`. Move về `apps/web/.env.local` là natural.
- **Related**: BUG-001 (`908c37e`) hé lộ rằng compose không đọc root `.env.local` mặc định — đã fix bằng `--env-file` flag, nhưng CR-001 sẽ thay bằng `infra/docker/.env` (compose auto-load).

## Proposed change

### Before (current state)

```
<repo-root>/
├── .env.example        # single template — mixed layers
├── .env.local          # single file (gitignored) — VITE_*, API_*, POSTGRES_*, REDIS_*, SESSION_*, UPLOAD_*
├── apps/api/           # reads process.env (no dotenv loader in dev)
├── apps/web/           # Vite scans workspace parents → reaches root .env.local
└── infra/docker/       # compose reads root .env.local via --env-file (BUG-001 fix)
```

### After (proposed)

```
<repo-root>/
├── apps/api/
│   ├── .env.example    # API runtime only: API_PORT, NODE_ENV, DATABASE_URL, REDIS_URL,
│   │                   #                   SESSION_SECRET, SESSION_COOKIE_NAME, COOKIE_SECURE,
│   │                   #                   SESSION_TTL_SECONDS, LOG_LEVEL, UPLOAD_DIR, UPLOAD_MAX_BYTES
│   └── .env.local      # gitignored
├── apps/web/
│   ├── .env.example    # FE runtime only: VITE_API_BASE_URL, VITE_APP_ORIGIN
│   └── .env.local      # gitignored
└── infra/docker/
    ├── .env.example    # infra only: POSTGRES_USER/PASSWORD/DB/PORT, REDIS_PORT
    └── .env            # gitignored (compose auto-load via project-dir)
```

**Loader strategy per layer**:

| Layer                      | How env is loaded                                                                                                                                                           |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `infra/docker/.env`        | Compose auto-load (project-directory default). Scripts giảm về `docker compose -f infra/docker/docker-compose.yml ...` — **không** cần `--env-file` flag nữa.               |
| `apps/api/` dev runtime    | Add `import "dotenv/config"` (hoặc native `node --env-file=.env.local` với Node 20.6+) ở `src/index.ts`. Drizzle scripts (`db:migrate` / `db:seed`) cần loader tương đương. |
| `apps/api/` tests          | `vitest.config.ts` đổi path từ `../../.env.local` → `./.env.local`.                                                                                                         |
| `apps/web/` dev + build    | Vite auto-load `apps/web/.env.local` (zero config).                                                                                                                         |
| `apps/web/` tests (vitest) | Không cần env — FE tests mock API qua MSW.                                                                                                                                  |

**Naming convention**: chọn `.env.local` ở `apps/*/` (match Vite convention), `.env` ở `infra/docker/` (match Docker Compose convention). Không dùng `.env.infra` / `.env.api` / `.env.web` ở root — tránh workspace-root clutter.

## Decision

**Approved** on 2026-04-23. User picks from §Open items:

1. File name: `apps/api/.env.local` + `apps/web/.env.local` (Vite convention).
2. `infra/docker/.env` — single file, không split dev/ci.
3. API env loader: **`dotenv` npm package** (not Node `--env-file` native).
4. Migration: **`scripts/migrate-env.sh`** tự split existing root file.

Implementation commits tuần tự:

1. `chore(infra): add infra/docker/.env.example + update docker scripts (CR-001)`
2. `chore(api): add .env.example + dotenv loader in dev + drizzle scripts (CR-001)`
3. `chore(web): add apps/web/.env.example (CR-001)`
4. `chore: remove root .env.example + update .gitignore (CR-001)`
5. `test(infra): update regression test to infra/docker/.env path (CR-001 / BUG-001)`
6. `docs: rewrite env setup flow across SETUP.md / CLAUDE.md / conventions-be.md / 03-architecture.md (CR-001)`
7. `docs: sync progress markers after CR-001 land`
