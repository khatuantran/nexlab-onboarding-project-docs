# CR-006 — UI Redesign: Dark vivid + glassmorphism

> Originally titled "Notion warm-friendly + graphics-rich" — superseded after v3.1 user feedback. See §Iteration v4 below.

## Summary

Redesign AppHeader + HomePage theo direction **Notion / Anthropic warm-friendly + graphics-rich** sau khi CR-005 (geometric patterns + multi-accent palette) bị reject 2 lần. New direction: warm canvas bg + sage secondary accent, solid filled buttons / banners, prominent icons (24-48px) + emoji decoration, minimal text (drop descriptive paragraphs). Pilot-first qua HomePage + AppHeader; sau approve mới roll out các màn khác.

## Motivation

CR-005 pilot v1 (`74c5855`) shipped heavy GradientMesh hero + BlobBackdrop + left-border accent strip — user reject "không match expected design" và cung cấp mockup target. CR-005 pilot v2 (`48420c6`) align với mockup (top accent strip + section header + Lọc dropdown + accent palette per project) — user reject lần 2 "xấu quá, thiết kế lại hoàn toàn, kể cả header".

User direction lock 2026-05-15 (round 3):

1. **Revert toàn bộ** 10 commit CR-005 (paperwork + foundation + cả 2 pilot). Reverted ở commit `1b2f863`.
2. **Visual direction**: Notion / Anthropic warm-friendly (chọn từ 3 option). Drop multi-accent palette + geometric patterns. Warm canvas bg + sage secondary.
3. **Graphics-rich constraint** (CRITICAL):
   - Nhiều icon / hình vẽ (prominent 28-48px, không chỉ 16px inline).
   - Button + banner fill màu solid (không outline-only).
   - Icon đặc sắc (striking lucide + emoji decoration).
   - Ít chữ nhiều hình — drop descriptive paragraphs ở hero / cards / empty / stat boxes; ưu tiên icon + color block.
4. **Layout**: top nav + full-width (kept structural). AppHeader simpler single-row, search prominent, icon-only actions. HomePage card grid graphics-rich (không phải list view).

CR-005 đã ship dù foundation tốt (accent tokens, pattern primitives) — accept loss per user explicit "bỏ hết". Có thể re-introduce sau nếu need, không invest upfront.

## Proposed change

### Before (post-revert state)

- AppHeader v2 chrome (2-row sticky: Row 1 logo + nav + search + bell + theme + user; Row 2 breadcrumb + admin CTA).
- HomePage: hero panel với eyebrow + h1 + subtitle paragraph, 3 StatChip horizontal tinted /10%, filter pill row + sort, 2-col ProjectCard grid với avatar + title + description + progress + footer (avatar stack + count + relative time).
- Visual-language charter v2 (CR-002): primary orange + secondary gold + neutral; gradient hero panel; hover rewards; lucide icons /40% empty states.
- Design-system: existing primary ramp + secondary gold + semantic (success/warning/info/destructive). Không có sage hoặc warm canvas.

### After (CR-006 pilot)

- **Charter v3 (new content)**: drop multi-accent + pattern primitives (those were CR-005 — reverted). Introduce warm canvas + sage accent, solid filled button/banner pattern, prominent icon policy, "ít chữ nhiều hình" density rule, friendly tone & copy guidelines.
- **Design-system update**: add `--canvas` (warm off-white) + `--sage` (secondary accent) tokens với light + dark variants. KHÔNG accent palette / pattern primitives.
- **AppHeader rewrite** (`.specs/ui/app-header.md` new spec): single-row sticky chrome, h-14, warm bg variant, brand mark + (optional) breadcrumb compact left + SearchInput prominent middle + icon-only Bell/Theme/UserMenu right. Row 2 ELIMINATE; admin CTA moves to HomePage section header.
- **HomePage rewrite**: hero compact (h1 "Góc onboarding 👋" only, no subtitle); 3 stat boxes (3-col grid) với solid filled icon plates 48×48 + value 32px + label uppercase; section header "Dự án" + Lọc outline + solid-filled "+ Tạo dự án mới" CTA; **card grid graphics-rich** (`sm:grid-cols-2 xl:grid-cols-3`) với mỗi card có **filled banner top** (bg-primary hoặc bg-sage, 64-80px tall, chứa icon prominent + project letters white) + body title + 2 metric chips (no description); admin Create tile dashed + large `+` icon; empty state emoji-led minimal.
- **ProjectCard rewrite** (graphics-rich banner, NOT list item): drop chevron + avatar-in-row layout; new pattern banner top + content body bottom.

## Iteration v4 — Direction reversal: Dark vivid + glassmorphism (2026-05-16, current pilot)

### Trigger

User exported 8 detailed screen mockups + tokens.css + app.css + topbar.jsx + nx.jsx + icons.jsx + data.jsx + design-system reference at [screens/dev's onboarding portal/](../../screens/dev%27s%20onboarding%20portal/) and prompted: "tôi đã update lại design system và các màn hình trong folder screens/screens hãy update vào docs và sửa lại các màn hình đúng như thiết kế mới".

The new direction is **opposite of v3 / v3.1 "Notion warm + Ít chữ nhiều hình"**:

- **Dark vivid hero** — linear-gradient 145deg `#0B0520 → #180E3A → #0C1A3A → #0A0E16` with 3 radial color blobs (orange / purple / blue) + dot-grid overlay + logo watermark mask.
- **Multi-accent palette REINSTATED** — 5 accent hues (purple #8B5CF6, green #10B981, blue #3B82F6, rose #F43F5E, amber #F59E0B) used in card gradients, role badges, section nav, integration cards.
- **Glassmorphism** — bg-white/{6,10,15,20} + backdrop-blur on cards layered over dark hero.
- **Gradient text headlines** — h1 with gradient span (`linear-gradient(90deg, #F07613 → #FFD092)`).
- **6-gradient ProjectCard** — header h-150px with one of 6 vivid gradients (orange-700→500, purple, green, blue, rose, amber) + decorative circles + initials plate + tag pill + Live pill + ProgressRing 52px.
- **ProgressRing primitive** — SVG circular progress (used in card header, feature tile header, feature detail).
- **GradientHero primitive** — reusable dark hero wrapper with blob slots + grid overlay + watermark slot.
- **AppHeader v4** — h-64 single row with NxLogo lockup + divider + search pill 400×38 with ⌘K kbd + Bell red-dot + Theme + UserMenu (initials gradient + name + role).
- **Floating filter bar** — `-mt-22` over hero edge with search + filter chip group (active = bg-primary text-white) + Tạo mới CTA.

### User decisions 2026-05-16

1. **CR strategy**: amend CR-006 with v4 block (keep CR ID, not new CR-007). v3.1 archived as superseded above.
2. **Scope**: pilot HomePage only (+ foundation tokens + AppHeader chrome). STOP for user review before rolling out remaining 7 screens (Login / ProjectLanding / FeatureDetail / Profile / Users / NEW Settings / Search).
3. **Dark mode**: keep Tailwind `.dark` class (reference uses `[data-theme="dark"]` but we don't switch — only an attribute name difference).
4. **Tokens cleanup**: drop CR-006 v3 / v3.1 lineage entirely (`--canvas*`, `--sage*`, `--tile-*`) — both light + dark.
