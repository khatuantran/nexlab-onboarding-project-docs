## Summary

As a **watcher of projects + features**, I want **NotificationBell trên AppHeader hiển thị unread count + dropdown danh sách thông báo** khi có người edit content tôi theo dõi, so that **bell button không còn `disabled` với tooltip "Thông báo (sắp ra mắt)" — actually delivers signal**.

## Motivation

NotificationBell shipped trong CR-006 v4 AppHeader v4 as a disabled placeholder (`tooltip "Thông báo (sắp ra mắt)"`). Largest mock surface by scope (full feature including pubsub-ish delivery). Triaged P3 vì cost cao + chỉ giá trị khi có watchers (BL-007) + activity (BL-005).

## Proposed scope

**In scope**:

- Schema: `notifications (id UUID, user_id UUID FK, kind TEXT, scope_type TEXT, scope_id UUID, payload JSONB, read_at TIMESTAMPTZ NULL, created_at TIMESTAMPTZ)`.
- BE: on section edit (PUT `/sections/...`), look up `watchers` for the parent feature + project, insert notification rows for each watcher (excluding the actor). Pino structured log shadow.
- API: `GET /api/v1/notifications?status=unread|all&limit=50` paginated; `POST /api/v1/notifications/:id/read`; `POST /api/v1/notifications/read-all`; `GET /api/v1/notifications/count` (unread count for bell badge).
- FE: NotificationBell enabled. Polls `/count` every 30 s when window active. Click → dropdown lists last 10 unread; "Đã đọc tất cả" + per-row read action. Real-time via polling only (no WebSocket v1).

**Out of scope**:

- WebSocket / SSE push.
- Email digest (defer to BL-NNN later).
- Per-channel preferences (mute project, mute kind).
- Inline-in-feed annotation (we're not building Discord).
- Push notification (browser native).

## Acceptance hint

- **Given** user A watches project P, **When** user B edits a section in P, **Then** within next poll user A's bell shows `count=1`.
- **Given** bell dropdown open, **When** click a notification, **Then** API marks read + link navigates to feature/section.

## Dependencies

- **Blocked by**: BL-007 watchers (defines "who gets notified") + BL-005 activity feed (defines edit event shape).
- **Blocks**: nothing (downstream nice-to-haves).
