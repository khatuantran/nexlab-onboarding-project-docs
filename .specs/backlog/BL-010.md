## Summary

As an **authenticated user**, I want **the 3 secondary tabs on `/projects/:slug` (Activity / Members / Settings) to render real content** instead of "Đang phát triển trong v2" empty state, so that **`components/projects/ProjectTabs.tsx` placeholder items become useful project-scoped views**.

## Motivation

CR-006 v4 ProjectTabs ships 4 tabs: "Tài liệu" (real catalog), then 3 placeholders. Each placeholder tab renders `PLACEHOLDER_ITEMS` with empty state copy. Visible across every project page. Triaged P2 — Members + Activity wait on BL-007 / BL-005 infrastructure; Settings tab overlaps with US-004 EditProjectDialog (already shipped).

## Proposed scope

**In scope** (split into 3 sub-stories when promoted):

- **Activity tab** — project-scoped feed (depends BL-005 endpoint extended to project scope or net-new `GET /projects/:slug/activity`).
- **Members tab** — list contributors (depends US-011 contributors derivation) + optional explicit-member assignment if membership table added (out of scope v1).
- **Settings tab** — promote existing `ProjectActionsMenu` actions (Edit / Archive) into a tab-resident form for admin discoverability. Low effort (mostly re-host existing components).

**Out of scope**:

- Explicit project membership / per-project ACL (vision: open access v1 per ADR).
- Comments / discussion thread.
- Project-level integration linking (covered by BL-009 Tích hợp panel).

## Acceptance hint

- **Given** user opens project, **When** clicks "Activity" tab, **Then** chronological feed of section edits in this project renders with cursor pagination.
- **Given** admin opens project, **When** clicks "Settings" tab, **Then** form mirrors `EditProjectDialog` + archive button inline.
