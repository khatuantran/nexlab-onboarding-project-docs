## Summary

As an **authenticated user**, I want **to add, remove, and reorder skill tags on my profile** (free-text + colored chip per skill), so that **`SkillsCard` trên `/profile` không hiển thị 7 hardcoded fake chips ("Business Analysis", "User Story", "Figma", "SQL", "BPMN", "Product Thinking", "Onboarding Doc") nữa**.

## Motivation

ProfilePage v4.1 (`31f8d3c`) ships `SkillsCard` với array `SKILLS` hardcoded 7 chip + cố định màu accent. User sees the same 7 fake skills on every account → no personal identity. Surfaced trong FE mock audit (CR-006 v4.6) as P2 because: (a) free-text skills are low-value-per-effort vs Profile fields (US-010) and contributors (US-011), (b) skill normalization / taxonomy is non-trivial if we don't want skill chaos.

## Proposed scope

**In scope** (when promoted):

- Migration: new table `user_skills (user_id UUID FK, label TEXT, color TEXT, sort_order INT, PRIMARY KEY(user_id, label))` OR simpler `users.skills JSONB` column.
- API: `GET /me` augment với `skills: SkillTag[]`. `PUT /me/skills` accept full array (replace-all semantics — simplest).
- FE: `SkillsCard` reads `user.skills` thay hardcoded. "Thêm" outline-dashed button opens small dialog to add a skill (label + color picker). "X" on chip to remove.

**Out of scope**:

- Skill taxonomy / autocomplete dropdown — free text only v1.
- Skill endorsement / voting — out.
- Skills on other-user view (profile pages for teammates) — only own profile renders skills v1.
- Skill search / filter — out.

## Acceptance hint

- **Given** user adds 3 skills, **When** they reload `/profile`, **Then** 3 chips persist in DB and render in same order.
- **Given** user has 0 skills, **When** SkillsCard renders, **Then** only the dashed "Thêm" button visible (no fake skills).

## Dependencies

- **Blocked by**: US-010 ship first (profile schema migration framework + EditProfileDialog pattern proven).
- **Blocks**: nothing (SkillsCard is its own surface).

## Promotion criteria

- [ ] Decide schema: separate `user_skills` table vs `users.skills` JSONB. Trade-off: JSONB simpler v1, normalization defer.
- [ ] Color palette restricted to the 7 v4 accent hues (purple/orange/green/blue/rose/amber/primary) or free hex picker?
- [ ] Max skills per user (e.g. cap 12)?
- [ ] User feedback: ≥ 2 user explicitly request skills feature (currently inferred from "tránh mock data").
