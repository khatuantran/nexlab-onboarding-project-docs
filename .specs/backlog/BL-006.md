## Summary

As a **user (own profile) or admin (own project)**, I want **to upload a wide cover image** for my profile header and (optionally) a project's GradientHero, so that **the "Đổi ảnh bìa" placeholder button on `/profile` (and the equivalent on ProjectHero) replaces with a real upload flow + persisted Cloudinary URL**.

## Motivation

ProfilePage v4.1 ships a "Đổi ảnh bìa" button wired to `placeholderToast`. ProjectHero v4 uses `GradientHero` primitive (no cover image; gradient + blobs only) but the v4.1 spec contemplates a per-project hero treatment. Both currently default to a generic gradient → no personality per profile / project.

## Proposed scope

**In scope**:

- Schema: `users.cover_url TEXT NULL`, `projects.cover_url TEXT NULL`.
- API: `POST /api/v1/me/cover` (mirror US-009 avatar pattern; Cloudinary folder `onboarding-portal/<env>/covers/users/`) + `POST /api/v1/projects/:slug/cover` (admin gate; folder `covers/projects/`).
- FE: Profile cover replaces gradient bg with `<img>` when `cover_url` set. Same for ProjectHero. Upload dialog reuses US-009 `AvatarUploadDialog` shape.

**Out of scope**:

- Cover image cropping UI — Cloudinary auto-transform (aspect-ratio `21/9`, fit `fill`) is enough.
- Cover gallery / preset library.
- Per-feature cover image.
- Animated covers (video / GIF).

## Acceptance hint

- **Given** user uploads `cover.jpg` ≤ 4 MB, **When** confirm, **Then** Cloudinary stores it + DB `users.cover_url` set + profile hero shows the image.
- **Given** admin uploads project cover, **When** any user views the project, **Then** ProjectHero renders the image (overlay still shows blobs + content for legibility).

## Dependencies

- **Blocked by**: nothing (US-009 avatar pattern is the template; CR-004 Phase 2 Cloudinary infra already shipped).
- **Blocks**: nothing.
