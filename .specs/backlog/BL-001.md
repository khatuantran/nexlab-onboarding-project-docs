## Summary

As an **author / admin**, I want **to attach a Git repo URL to each project and a Pull Request URL to each feature**, so that **the "Repo" button on `/projects/:slug` and "Xem PR" button on `/projects/:slug/features/:fSlug` become real links instead of placeholder toasts**.

## Motivation

CR-006 v4 ProjectHero shipped a "Repo" ghost-glass button + FeatureDetail shipped a "Xem PR" button, both wired to `placeholderToast("...: tính năng đang phát triển trong v2")`. Today users see a toast nudging them to v2 — feels half-done. Linking to existing GitHub repos/PRs is trivial DB work (2 nullable text columns) but blocked on the fact that `EditFeatureDialog` doesn't exist yet (US-008 archive-only landed; feature edit deferred).

## Proposed scope

**In scope** (when promoted):

- Migration: `projects.repo_url TEXT NULL`, `features.pr_url TEXT NULL`.
- API: `PATCH /projects/:slug` accept `repoUrl?: string | null`. New or existing `PATCH /features/:id` accept `prUrl?: string | null`. URL format validate (start with `https://`).
- FE: `EditProjectDialog` extend with `repoUrl` input (existing dialog — easy). Build `EditFeatureDialog` (NEW) with `title + slug + prUrl` fields, triggered from `FeatureActionsMenu` "Sửa feature" item (US-008 menu has archive-only today). Repo / Xem PR buttons become `<a href={url} target="_blank" rel="noopener">` when url present; render disabled placeholder when null.

**Out of scope**:

- GitHub OAuth / API integration (no fetch PR status, reviewers, etc.) — pure URL store + click-through.
- Repo branch / commit linking — top-level repo URL only.
- Per-section PR linking — feature-level only.

## Acceptance hint

- **Given** admin edits project, **When** they set `repoUrl = "https://github.com/foo/bar"`, **Then** "Repo" button on landing becomes a real link opening in new tab.
- **Given** feature has no `prUrl`, **When** "Xem PR" rendered, **Then** button disabled với muted tooltip "Chưa link PR".

## Dependencies

- **Blocked by**: EditFeatureDialog component doesn't exist yet (US-008 shipped archive only). Need a small story or sub-task to build it before this backlog item can promote.
- **Blocks**: Real "Repo" / "Xem PR" affordances in v4 UI — currently dead buttons.

## Promotion criteria

- [ ] EditFeatureDialog shipped (or scoped as part of this BL).
- [ ] URL format validation rules confirmed (`https://` prefix + max 500 chars).
- [ ] Decision on whether to validate domain (github.com only?) or accept any URL.
