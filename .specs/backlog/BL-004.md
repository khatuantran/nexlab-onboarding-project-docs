## Summary

As an **authenticated user**, I want **my `/profile` "Recent projects" card to show projects I myself recently edited** (ranked by my latest `sections.updated_at` on each), so that **the 4 hardcoded entries (`["US-3 E2E", "E2E Backend", "Catalog Search", "Momo Payment"]` with fake 80/50/40/100 percentages) get replaced with truth**.

## Motivation

ProfilePage v4.1 (`31f8d3c`) ships `RecentProjectsCard` with 4 hardcoded rows. Each user sees the same fake projects. Surfaced in CR-006 v4.6 mock audit (P1 — useful "jump back in" signal once real).

## Proposed scope

**In scope**:

- New endpoint `GET /api/v1/me/recent-projects?limit=4` returning `[{ slug, name, lastTouchedAt, sectionsTouched }]` sorted by `lastTouchedAt` desc, derived from `sections.updated_by = me`.
  - Reuses repo pattern from US-011 contributors (`projectRepo.getContributorsForProjects`) inverted: group by user, sort by max(updated_at) per project.
- FE: ProfilePage `RecentProjectsCard` reads from `useMyRecentProjects()` hook. Each row = project slug link + name + my latest edit time + per-project section-touch count badge.

**Out of scope**:

- Per-project `pct` filled — that needs `filledSectionCount` aggregation (separate BL).
- Pinning / favoriting projects.
- "More from this project" lateral nav.

## Acceptance hint

- **Given** user touched projects A, B, C (latest in that order), **When** GET /me/recent-projects?limit=4, **Then** array `[A, B, C]` (3 items, sorted by recency desc, capped at 4).
- **Given** brand-new user, **When** GET /me/recent-projects, **Then** empty array. FE renders "Chưa có hoạt động gần đây" empty state.

## Dependencies

- **Blocked by**: US-011 ship first (shares the `sections.updated_by`-aggregation infrastructure).
- **Blocks**: nothing (BL-005 activity feed could reuse the same pattern but isn't gated).

## Promotion criteria

- [ ] Decision: separate endpoint vs extend `GET /me`. (Recommend: separate; `/me` should stay slim.)
- [ ] Decide limit default (4 visible per card · 10 server cap).
- [ ] Confirm "section touched" = updated_by match (excludes unchanged sections initial seed by another user).
