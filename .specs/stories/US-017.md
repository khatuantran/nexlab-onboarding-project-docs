## User perspective

> As an **authenticated user**, I want **a real activity feed on `/profile` AND a usable "Xem tất cả" drawer on FeatureDetail's ActivityRail**, so that **the 4 hardcoded rows + `placeholderToast("Lịch sử hoạt động đầy đủ...")` get replaced with truth**.

## Scope in

- New endpoint `GET /api/v1/me/activity?limit=20&cursor=<iso>` (auth) returning paginated `{ items: ActivityItem[], nextCursor: string | null }`.
  - `ActivityItem = { id, sectionType, featureSlug, featureTitle, projectSlug, projectName, updatedAt, verbCode: 'updated' }`.
  - Cursor format: ISO timestamp of last item's `updated_at`. `WHERE updated_at < $cursor ORDER BY updated_at DESC LIMIT $limit + 1`; if length === limit+1 pop last + emit `nextCursor`.
  - Limit default 20, max 50.
  - Source: `sections` table where `updated_by = me`. Exclude archived features + projects. Verb fixed `'updated'` v1.
- FE: `ProfilePage` `ActivityFeedCard` reads `useMyActivity()` (top 20, no pagination shown on profile — just first page).
- FE: `ActivityRail` "Xem tất cả" → shadcn `Sheet` drawer (right-side). Drawer shows ALL 5 sections of current feature sorted by `updated_at` desc (no new fetch — reuses `sections` prop). Drop `placeholderToast`. Drawer footer hint "Lịch sử chi tiết hơn sẽ ra mắt sau."

## Scope out

- Feature-scoped `/projects/:slug/features/:fSlug/activity` endpoint — **dropped**: feature has only ≤5 section rows (PUT overwrites), so pagination is moot. Drawer reuses already-loaded sections.
- Per-project activity feed (BL-010 ProjectTabs).
- Real-time updates.
- Verb types beyond `'updated'`.

## Acceptance criteria

### AC-1 — /me/activity shape

GET /me/activity → 200 `{ data: { items: ActivityItem[], nextCursor: string | null } }`.

### AC-2 — chronological order

**Given** user has 3 edits in 3 distinct sections at T-1d, T-2h, T-30m **When** GET /me/activity **Then** items[0] = T-30m, items[2] = T-1d.

### AC-3 — cursor pagination

**Given** user has 25 edits **When** GET /me/activity?limit=20 **Then** items.length===20, nextCursor = updatedAt of items[19] (ISO string). 2nd call with that cursor returns remaining 5; nextCursor = null.

### AC-4 — archive excluded

Archived feature / project edits NOT in feed.

### AC-5 — empty

New user → `{ items: [], nextCursor: null }`. FE shows "Chưa có hoạt động — bắt đầu chỉnh sửa để xuất hiện ở đây".

### AC-6 — limit cap

?limit=100 clamps to 50; ?limit=0 → 400 VALIDATION_ERROR; default 20.

### AC-7 — 401 unauth

### AC-8 — FE ProfilePage feed

ActivityFeedCard reflects API rows (verb-translated to "cập nhật") with relative time.

### AC-9 — FE ActivityRail drawer

"Xem tất cả" opens right-side sheet showing all 5 section rows (empty body shown as italic "Chưa có nội dung"); body sorted by updated_at desc (NULL last). No placeholder toast.
