## User perspective

### As a Reader (Dev mới onboard)

> As a **Reader**, I want **gõ vài ký tự đầu của project name (ví dụ `"a"`) để thấy `"a3solutions"` ngay**, so that **tôi không phải nhớ + gõ full token mỗi lần search**.

### As a BA / PM

> As a **BA**, I want **search bằng tiếng Việt không dấu (`"dang nhap"`) match được nội dung có dấu (`"Đăng nhập"`)**, so that **tôi gõ nhanh trên bàn phím US-layout mà vẫn ra kết quả**.

### As a Senior Dev

> As a **Tech Lead**, I want **typo nhẹ (`"ondoarding"`) vẫn match `"onboarding"`**, so that **tôi không bị frustration khi search tên dự án/feature**.

## Why this story now?

US-005 ship grouped multi-entity search 2026-04-25, dùng `plainto_tsquery('simple', q)` exact-token matching. Sau pilot 2026-04-26+, user feedback chỉ ra 3 friction:

- Partial token không match (gõ `"a"` không ra `"a3solutions"`).
- Vietnamese không dấu không match nội dung có dấu.
- Typo nhẹ làm zero-result.

US-005 §Scope out: "Typo tolerance / fuzzy match → defer (BL-NNN: Meilisearch evaluation)". Story này revisit decision đó với approach **Postgres-only** (built-in `unaccent` + `pg_trgm`), giữ $0 infra constraint per [ADR-002](../adr/ADR-002-deployment-platform.md).

## Scope in

- **DB migration** `0005_search_unaccent_trgm.sql`:
  - Enable `unaccent` + `pg_trgm` extensions.
  - Wrap `unaccent` thành `immutable_unaccent(text) RETURNS text IMMUTABLE` (Postgres `unaccent` mặc định `STABLE`, cần immutable cho generated columns + functional indexes).
  - Rebuild `projects.search_vector` + `uploads.search_vector` (generated columns) dùng `immutable_unaccent`.
  - Update `features_rebuild_search_vector` trigger function dùng `immutable_unaccent`. Backfill existing features.
  - Trigram GIN indexes: `projects.name`, `features.title`, `users.display_name`, `uploads.filename` (unaccented).
- **Backend** (`apps/api/src/repos/searchRepo.ts`):
  - Helper `buildTsQuery(q: string): string` — sanitize → alphanumerics only → `to_tsquery('simple', '<token>:*')` per token, joined `&`.
  - 5 entity queries switch `plainto_tsquery` → `to_tsquery(buildTsQuery(q))`.
  - Trigram fallback per "short field" entity (projects/features/authors/uploads): `WHERE search_vector @@ tsquery OR immutable_unaccent(field) % immutable_unaccent($q)`.
  - Ranking: `greatest(ts_rank(...), similarity(immutable_unaccent(field), immutable_unaccent($q)))`.
  - `ts_headline` vẫn dùng same tsquery → highlight đúng.
- **Author search**: thay raw `ILIKE` bằng `immutable_unaccent(display_name) ILIKE immutable_unaccent($q || '%')` + trigram rank.
- **Tests**:
  - Unit BE: prefix match cases (5 entity), unaccent cases, trigram fallback cases.
  - Integration BE: route returns `data.projects[0].slug = "a3solutions"` cho `q=a`.
  - E2E: extend `e2e/us-005.spec.ts` thêm prefix scenario.

## Acceptance criteria

### AC-1 — Prefix match project name

**Given** project có `name = "A3Solutions"`
**When** user submit `GET /api/v1/search?q=a`
**Then** `data.projects[0].slug = "a3solutions"`
**And** `snippet` chứa `<mark>A</mark>3Solutions` hoặc tương đương.

### AC-2 — Prefix match feature title

**Given** feature có `title = "Đăng nhập SSO"` thuộc project bất kỳ
**When** user submit `q=đăn`
**Then** `data.features[0].featureSlug` match feature đó.

### AC-3 — Accent-insensitive Vietnamese

**Given** project name `"Đăng nhập"`
**When** user submit `q=dang nhap` (không dấu)
**Then** `data.projects[0]` match project đó
**And** ngược lại, `q=đăng nhập` match content được index `"Dang nhap"` (không dấu).

### AC-4 — Trigram fuzzy fallback (typo)

**Given** project name `"onboarding"` và query `q=ondoarding` (sai 1 ký tự)
**When** user submit
**Then** `data.projects` chứa project `"onboarding"` qua trigram similarity ≥ 0.3
**And** rank của trigram-only hit < rank của tsquery hit cùng query.

### AC-5 — Author display name accent-insensitive

**Given** user `display_name = "Lê Văn Hùng"`
**When** user submit `q=hung`
**Then** `data.authors[0].displayName = "Lê Văn Hùng"`.

### AC-6 — No regression existing US-005 AC

**Given** existing US-005 AC-1..AC-15 fixtures
**When** test suite chạy
**Then** mọi AC vẫn pass (response shape, filters, archived exclude, sanitize, deep-link đều không đổi).

### AC-7 — Sanitize tsquery input

**Given** query chứa special chars `q=foo&|!bar`
**When** server processes
**Then** **không** raise `to_tsquery` syntax error
**And** treat input như alphanumerics only (`foo bar` → 2 prefix tokens).

## UX notes

- **Zero FE change**: improvement transparent. SearchPage layout, FilterBar, result cards, URL state đều không đụng.
- **Snippet rendering**: vẫn DOMPurify allow-list `["mark"]` — XSS posture giữ nguyên.
- **Latency budget**: target ≤ 500ms p95 (NFR-PERF-001) unchanged. Trigram + tsquery union có thể chậm hơn pure tsquery — benchmark trong T5.

## Dependencies

- US-005 ship (searchRepo, route, FE pipeline).
- Postgres ≥ 12 (Neon: PG 16 — OK).
- Extensions `unaccent` + `pg_trgm` available trên Neon free tier (verified — both standard contrib modules).

## Downstream consumers

- **Future US** (deferred): Search-as-you-type autocomplete sẽ build trên prefix-match foundation.
- **M3 pilot scaling**: corpus tăng → fuzzy fallback cứu zero-result cases.

## Risks & open items

- **Generated column drop+recreate** trên `projects` + `uploads`: cần backfill — pilot < 100 row OK, nhưng nếu sau prod scale, cần lock + offline migration. Document trong runbook.
- **`to_tsquery` syntax errors** với special chars: mitigation = `buildTsQuery` strip non-alphanumeric trước append `:*`. AC-7 cover.
- **Trigram threshold 0.3**: empirical default. Có thể cần tune theo corpus thực tế. Document trong PR comment.
- **`ts_headline` highlight với prefix tsquery**: chưa verify edge case (prefix marker `:*` có thể không highlight đúng partial). Test riêng trong T4.
- **Performance**: 5 query × (tsquery OR trgm) có thể tăng latency ~30%. Benchmark trong T5; nếu vượt budget, drop trgm fallback cho features (giữ projects/authors/uploads).
- **Index size**: 4 trigram GIN index thêm. Pilot scale OK, document trong NFR-PERF-001 risk register.
