## User perspective

> As an **author / admin**, I want **to attach a Git repo URL to each project and a Pull Request URL to each feature**, so that **"Repo" button trên `/projects/:slug` và "Xem PR" button trên `/projects/:slug/features/:fSlug` mở đúng GitHub / GitLab / Bitbucket link thay vì placeholder toast "v2"**.

## Why this story now?

CR-006 v4 ProjectHero ship "Repo" ghost-glass button + FeatureDetail ship "Xem PR" — cả 2 hiện wire `placeholderToast(...)`. Mock surface visible nhất sau US-010/US-011. Blocked on US-012 (EditFeatureDialog) — đã ship. Effort thấp: 2 nullable text columns + extend 2 PATCH endpoint + extend 2 dialog với 1 field URL mới.

## Scope in

- **Schema**: migration `0011_projects_repo_url_features_pr_url.sql`:
  ```sql
  ALTER TABLE projects ADD COLUMN repo_url TEXT;
  ALTER TABLE features ADD COLUMN pr_url TEXT;
  ```
- **Shared**:
  - `urlSchema = z.string().regex(/^https?:\/\//).max(500)` exported từ `packages/shared/src/schemas/common.ts` (mới hoặc colocate).
  - `updateProjectRequestSchema` extend với `repoUrl: urlSchema.nullable().optional()`.
  - `updateFeatureRequestSchema` (US-012) extend với `prUrl: urlSchema.nullable().optional()` + refine vẫn yêu cầu ≥ 1 key.
  - `ProjectResponse` + `ProjectSummary` thêm `repoUrl: string | null`.
  - `FeatureResponse` + `FeatureListItem` thêm `prUrl: string | null`.
- **API**:
  - `PATCH /projects/:slug` accept `repoUrl?: string | null` (missing → unchanged; null → clear; string → update).
  - `PATCH /projects/:slug/features/:featureSlug` accept `prUrl?: string | null`.
  - Response shape: tất cả endpoint trả project / feature giờ include `repoUrl` / `prUrl`.
- **FE**:
  - `EditProjectDialog` thêm input "Repo URL" (optional, placeholder `https://github.com/...`).
  - `EditFeatureDialog` (US-012) thêm input "PR URL".
  - `ProjectHero` Repo button: `repoUrl ? <a href={repoUrl} target="_blank" rel="noopener noreferrer">` : `<button disabled tooltip="Chưa link repo">`.
  - `FeatureDetailPage` Xem PR button: same pattern với `prUrl`.

## Scope out

- **GitHub API integration** — không fetch PR status, reviewers, merge state. Pure URL store.
- **Repo branch / commit linking** — top-level URL only.
- **Per-section PR linking** — feature-level only.
- **Domain whitelist** — accept bất kỳ `http(s)` URL (user pick "Any URL" trong triage 2026-05-16).
- **Auto-extract repo từ feature PR** — separate.
- **URL preview cards / OG fetch** — out (FR-EMBED-001 đã cover client-side render cho 3 domain hardcoded).

## Acceptance criteria

### AC-1 — PATCH /projects accepts repoUrl

**Given** admin authenticated, project `pilot` tồn tại
**When** PATCH `/projects/pilot` body `{ name: 'Pilot', repoUrl: 'https://github.com/foo/bar' }`
**Then** 200 response chứa `data.repoUrl === 'https://github.com/foo/bar'`
**And** F5 → DB row có `repo_url = 'https://github.com/foo/bar'`.

### AC-2 — PATCH /projects null clears repoUrl

**Given** project có `repo_url='https://...'`
**When** PATCH body `{ name: '...', repoUrl: null }`
**Then** DB `repo_url IS NULL`.

### AC-3 — PATCH /projects missing key preserves repoUrl

**Given** project có `repo_url='https://x'`
**When** PATCH body `{ name: 'New name' }` (không có `repoUrl`)
**Then** `repo_url` vẫn `'https://x'` (không bị clear).

### AC-4 — PATCH /projects rejects invalid URL

**Given** admin authenticated
**When** PATCH body `{ name: '...', repoUrl: 'not-a-url' }` (không start `http(s)://`)
**Then** 400 `VALIDATION_ERROR` field `repoUrl`.

### AC-5 — PATCH /features accepts prUrl

**Given** admin, feature `pilot/dang-nhap`
**When** PATCH body `{ prUrl: 'https://github.com/foo/bar/pull/42' }`
**Then** 200 với `data.prUrl` set.

### AC-6 — PATCH /features null clears prUrl + AC-7 missing key preserves

(Mirror AC-2 + AC-3 cho feature `prUrl`.)

### AC-8 — Project response shape includes repoUrl

**Given** project có `repo_url='https://github.com/foo/bar'`
**When** GET `/projects/pilot` hoặc `/projects` list
**Then** mỗi item chứa key `repoUrl` với giá trị tương ứng (null khi chưa set).

### AC-9 — Feature response shape includes prUrl

**Given** feature có `pr_url` set
**When** GET `/projects/pilot/features/dang-nhap` hoặc list trong project response
**Then** key `prUrl` xuất hiện.

### AC-10 — FE: Repo button renders link khi repoUrl set

**Given** `/projects/pilot` với `repoUrl='https://github.com/foo/bar'`
**When** page render
**Then** "Repo" button là anchor `<a href='https://github.com/foo/bar' target='_blank' rel='noopener noreferrer'>` (test assert `getByRole('link', { name: /repo/i }).getAttribute('href')`).

### AC-11 — FE: Repo button disabled khi repoUrl null

**Given** project có `repoUrl=null`
**When** page render
**Then** Repo button render `<button disabled>` với `aria-label` chứa "Chưa link repo" hoặc tooltip equivalent
**And** click → không navigate, không toast (no-op).

### AC-12 — FE: Xem PR button mirror AC-10/11 cho `prUrl`

(Same rules trên `FeatureDetailPage`.)

### AC-13 — FE: EditProjectDialog input "Repo URL"

**Given** admin mở "Sửa project"
**When** dialog mở
**Then** input "Repo URL" hiển thị, prefilled từ `project.repoUrl` (rỗng nếu null)
**And** điền → submit → PATCH body chứa `repoUrl: '...'`.

### AC-14 — FE: EditFeatureDialog input "PR URL"

(Mirror AC-13 cho dialog feature.)

## UX notes

- Input placeholder: `https://github.com/owner/repo` / `https://github.com/owner/repo/pull/123`.
- Helper text mute: "Bất kỳ link http(s), tối đa 500 ký tự."
- Empty input → submit as `null` nếu trước đó có value (explicit clear); giữ missing nếu pristine empty.
- Repo button disabled style: thay vì hide hoàn toàn, render disabled với tooltip → giảm layout shift.

## Risks & open items

- **No URL existence check** — accept dead link. Acceptable v1; user click sẽ thấy GitHub 404. Tooltip nói "Mở repo / PR ở tab mới".
- **`updated_by` audit** — same as US-012 (feature row không track). Out.
- **XSS qua URL** — render qua React `<a href>` không dangerouslySetInnerHTML; `target="_blank" rel="noopener noreferrer"` mandatory để tránh tab-napping.
- **PATCH project bắt buộc `name`** — schema hiện `name` required, sẽ giữ. Patch chỉ `repoUrl` cũng phải gửi `name` (acceptable, dialog luôn submit full form). Có thể relax thành optional sau nếu cần.
