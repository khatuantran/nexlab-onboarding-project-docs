## User perspective

> As an **authenticated user (own profile)** or **admin (own project)**, I want **to upload a wide cover image**, so that **`/profile` header và ProjectHero hiển thị ảnh cá nhân hoá thay vì gradient mặc định, và button "Đổi ảnh bìa" hiện đang `placeholderToast` được thay bằng flow upload thật**.

## Scope in

- **Schema**:
  - `users.cover_url TEXT NULL` — migration `0013_users_cover_url.sql`.
  - `projects.cover_url TEXT NULL` — same migration (single file 2 columns).
- **API** (2 endpoints, mirror US-009 avatar pattern):
  - `POST /api/v1/me/cover` — multipart `file` ≤ 4 MB (png/jpg/webp). `requireAuth`. Cloudinary folder `onboarding-portal/<env>/covers/users/`. Response `{ data: { coverUrl } }`.
  - `POST /api/v1/projects/:slug/cover` — same shape. `requireAuth + requireAdmin`. Cloudinary folder `onboarding-portal/<env>/covers/projects/`.
- **Read augment**: `GET /me` augments `AuthUser` with `coverUrl: string | null`. `GET /projects` / `GET /projects/:slug` augment `ProjectSummary` / `ProjectResponse` with `coverUrl: string | null`.
- **FE**:
  - `ProfilePage` header — hiện tại có button "Đổi ảnh bìa" gọi placeholderToast → wired to new `CoverUploadDialog`. Khi `coverUrl` set: render `<img>` với `object-cover` + overlay `bg-gradient-to-b from-black/40 to-black/60` để text rõ. Khi null: giữ gradient hiện tại.
  - `ProjectHero` — admin-only `CoverUploadDialog` (overflow menu hoặc inline button). Khi `coverUrl` set: replace `bg-gradient-to-br` của `GradientHero` bằng `<img>` + same overlay; blobs glow giữ nguyên cho v4 hero look.
  - Reuse pattern + dialog shape từ [AvatarUploadDialog](../../apps/web/src/components/profile/AvatarUploadDialog.tsx) (US-009).

## Scope out

- Cover image cropping UI — Cloudinary `c_fill,ar_21:9` auto-transform là đủ v1.
- Cover gallery / preset library.
- Per-feature cover image (FeatureDetailPage giữ gradient hiện tại).
- Animated covers (video/GIF/Lottie).
- Cleanup orphan Cloudinary covers khi user upload mới — accept storage drift (cùng pattern US-009).

## Acceptance criteria

### AC-1 — POST /me/cover happy path

**Given** authenticated user **When** POST multipart `file=cover.jpg` (2 MB, image/jpeg) **Then** 200 `{ data: { coverUrl: "https://res.cloudinary.com/..." } }` **And** `users.cover_url` row updated **And** subsequent `GET /me` returns `coverUrl` matching.

### AC-2 — POST /me/cover 4 MB cap

**Given** authenticated user **When** POST file 5 MB **Then** 413 `FILE_TOO_LARGE` ("File quá lớn (max 4 MB)").

### AC-3 — POST /me/cover mime whitelist

**Given** authenticated user **When** POST `file.pdf` (application/pdf) **Then** 415 `UNSUPPORTED_MEDIA_TYPE`.

### AC-4 — POST /me/cover 401

**Given** unauth **When** POST /me/cover **Then** 401 `UNAUTHENTICATED`.

### AC-5 — POST /projects/:slug/cover admin only

**Given** non-admin (role=author) **When** POST /projects/demo/cover **Then** 403 `FORBIDDEN`.

### AC-6 — POST /projects/:slug/cover happy path

**Given** admin **When** POST /projects/demo/cover with 3 MB jpg **Then** 200 `{ data: { coverUrl } }` **And** `projects.cover_url` updated **And** `GET /projects/demo` returns `coverUrl`.

### AC-7 — POST /projects/:slug/cover 404

**Given** admin **When** POST /projects/no-such/cover **Then** 404 `PROJECT_NOT_FOUND` — no Cloudinary upload happens.

### AC-8 — Cloudinary down

**Given** Cloudinary provider 5xx **When** any cover upload **Then** 502 `UPLOAD_PROVIDER_ERROR`.

### AC-9 — Cloudinary unconfigured

**Given** Cloudinary env vars missing **When** upload **Then** 503 `UPLOADS_DISABLED`.

### AC-10 — GET /me exposes coverUrl

**Given** user has uploaded cover **When** `GET /me` **Then** response `data.coverUrl` matches DB; `null` when never uploaded.

### AC-11 — GET /projects exposes coverUrl

**Given** project has cover **When** `GET /projects` or `GET /projects/:slug` **Then** response includes `coverUrl: string | null`.

### AC-12 — FE ProfilePage cover render

**Given** API returns `coverUrl` **When** ProfilePage mounts **Then** header renders `<img src={coverUrl} alt="" class="object-cover ...">` with `bg-gradient-to-b from-black/40 to-black/60` overlay; title/meta readable.

### AC-13 — FE ProfilePage empty state

**Given** API returns `coverUrl: null` **When** ProfilePage mounts **Then** header giữ gradient mặc định + button "Đổi ảnh bìa" visible.

### AC-14 — FE ProjectHero cover render (admin gate for button)

**Given** project has `coverUrl` **When** anyone views **Then** ProjectHero render `<img>` + overlay + giữ blobs glow. **Given** viewer is admin **Then** "Đổi ảnh bìa" button visible (overflow menu hoặc inline). Non-admin không thấy button.

## UX notes

- `CoverUploadDialog` (NEW) — clone shape `AvatarUploadDialog`:
  - File picker accept `image/png,image/jpeg,image/webp`.
  - Client-side validate ≤ 4 MB + mime trước khi POST.
  - Preview thumbnail aspect `21/9` (450×210 max).
  - Submit button "Lưu ảnh bìa" with `pending` state.
  - Toast success "Đã cập nhật ảnh bìa" + invalidate `["auth","me"]` (profile) or `["projects", slug]` (project).
- Overlay treatment khi có cover: `bg-gradient-to-b from-black/40 to-black/60` trên `<img>`. Blobs giữ glow để v4 hero identity còn nhận ra.
- Empty state khi `coverUrl: null`: giữ gradient hiện tại + button "Đổi ảnh bìa" (placeholderToast của ProfilePage được thay).

## Risks & open items

- **Cloudinary cost drift**: cover ảnh lớn hơn avatar (~4 MB vs ~2 MB) → bandwidth + storage tăng. Track post-pilot, nếu vượt $5/tháng → enforce smaller `c_limit,w_2000` transform.
- **Overlay legibility**: 40-60% black gradient có thể quá mạnh trên ảnh dark. Nếu user feedback → swap to `backdrop-blur` v2.
- **Aspect ratio mismatch**: user upload portrait → Cloudinary `c_fill,ar_21:9` crop trung tâm, có thể cut head. Accept v1; gallery/cropping defer to BL-006b.
