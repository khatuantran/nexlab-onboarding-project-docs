## User perspective

> As an **admin (and downstream, author)**, I want **to edit a feature's title and slug after creation**, so that **I can fix typos, rename a feature scope, or align the slug with new URL conventions without losing sections/uploads/contributors**.

## Why this story now?

FR-FEAT-001 đã spec "When an authenticated user edits a feature's title or slug, the system shall update it" nhưng cho tới M2 chỉ create + archive ship; edit slipped. v4 UI giới thiệu `FeatureActionsMenu` (US-008) chỉ có 1 item "Lưu trữ feature" — "Sửa feature" missing. Cũng là **blocker** cho US-013 (repo/PR URLs) vì dialog primitive cần tồn tại trước khi thêm field `prUrl`.

---

## Scope in

- API:
  - `PATCH /api/v1/projects/:slug/features/:featureSlug` accept `{ title?, slug? }`.
  - 200 → `{ data: FeatureResponse }`. 404 `FEATURE_NOT_FOUND`, 409 `FEATURE_SLUG_TAKEN`, 403 `FORBIDDEN` (non-admin).
  - Slug edit validates kebab-case + uniqueness trong cùng project; conflict same-feature (no-op rename) treated OK.
- Shared:
  - `updateFeatureRequestSchema` Zod object với `title?, slug?` cả 2 optional, ≥ 1 key bắt buộc.
- FE:
  - `EditFeatureDialog` component mới ở `apps/web/src/components/features/EditFeatureDialog.tsx`. Form: title + slug input, validate client-side (slug regex + uniqueness server-side).
  - `FeatureActionsMenu` thêm item "Sửa feature" (admin only) bên trên "Lưu trữ feature" với pencil icon.
  - Mutation hook `useUpdateFeature(projectSlug, featureSlug)` ở `queries/projects.ts` invalidates `["projects", projectSlug]` + `["projects", projectSlug, "features", featureSlug]`; navigate khi slug đổi.

## Scope out

- **`prUrl` field** — defer US-013 (kèm `repoUrl` migration).
- **Edit feature description** — feature chưa có `description` column.
- **Mass rename / batch slug edit** — out.
- **Slug history / redirect old → new** — hard 404 sau rename, document trong UX.
- **Author role edit access** — admin-only v1 (mirror US-004 project edit gate); promote to author trong v2 nếu pilot retro yêu cầu.

## Acceptance criteria

### AC-1 — PATCH updates title only

**Given** admin authenticated, feature `pilot/dang-nhap` tồn tại với `title='Đăng nhập'`
**When** PATCH `/projects/pilot/features/dang-nhap` body `{ title: 'Sign in' }`
**Then** response 200 `{ data: { title: 'Sign in', slug: 'dang-nhap', ... } }`
**And** F5 → feature row có `title='Sign in'`, `updated_at` bumped.

### AC-2 — PATCH updates slug only

**Given** feature `pilot/dang-nhap`
**When** PATCH body `{ slug: 'sign-in' }`
**Then** response 200 với `slug='sign-in'`
**And** GET `/projects/pilot/features/dang-nhap` → 404 (old slug dead)
**And** GET `/projects/pilot/features/sign-in` → 200.

### AC-3 — PATCH updates both fields atomically

**Given** feature exists
**When** PATCH body `{ title: 'New', slug: 'new-slug' }`
**Then** cả 2 thay đổi cùng 1 transaction; partial failure không leak (test bằng cách force conflict trong CI).

### AC-4 — PATCH with empty body rejected

**Given** admin authenticated
**When** PATCH body `{}`
**Then** 400 `VALIDATION_ERROR` với message "Cần ít nhất 1 trường".

### AC-5 — PATCH rejects slug conflict trong project

**Given** project có 2 feature `a, b`
**When** PATCH `a` body `{ slug: 'b' }`
**Then** 409 `FEATURE_SLUG_TAKEN`.

### AC-6 — PATCH non-admin rejected

**Given** authenticated user role=`author`
**When** PATCH bất kỳ feature
**Then** 403 `FORBIDDEN`.

### AC-7 — PATCH non-existent feature 404

**Given** admin
**When** PATCH `/projects/pilot/features/ma-khong-ton-tai` body `{ title: 'x' }`
**Then** 404 `FEATURE_NOT_FOUND`.

### AC-8 — FE: admin menu has "Sửa feature" item

**Given** admin trên `/projects/pilot` (HomePage feature card grid) hoặc `/projects/pilot/features/dang-nhap`
**When** click `FeatureActionsMenu` trigger
**Then** menu hiển thị 2 item: "Sửa feature" (pencil icon) + "Lưu trữ feature" (archive icon)
**And** author role chỉ thấy 0 item (menu hidden như cũ).

### AC-9 — FE: dialog submit happy path

**Given** admin click "Sửa feature" → `EditFeatureDialog` mở với title + slug prefilled
**When** đổi title → "New title" → click "Lưu"
**Then** PATCH sent với `{ title: 'New title' }`
**And** toast "Đã cập nhật feature"
**And** dialog close
**And** card refreshes với title mới.

### AC-10 — FE: dialog navigates on slug change

**Given** admin trên `/projects/pilot/features/dang-nhap` mở dialog
**When** đổi slug → "sign-in" → submit success
**Then** browser navigate sang `/projects/pilot/features/sign-in` (router.replace) thay vì 404.

## UX notes

- Dialog mirror style `EditProjectDialog` (US-004): shadcn `Dialog` + 2 `Input` stacked + footer "Huỷ / Lưu".

## Risks & open items

- **Slug rename = broken bookmarks** — Document trong UX helper; v2 có thể add slug history table.
