## User perspective

> As a **Tech Lead (Hùng)**, I want to **append tech-notes (markdown + embed link Jira/Figma/GitHub) và upload screenshots cho feature do BA đã author**, so that **dev mới đọc catalog thấy cả business lẫn implementation context trong 1 nơi**.

## Why after US-002?

US-002 đã có edit flow cho 3 business section. US-003 tái sử dụng editor layout + section PUT endpoint, chỉ thêm:
1. Embed preview card (FE render logic).
2. File upload endpoint + FE upload widget.

Validate multi-author model (per-section `updated_by`).

---

## Scope in

- Edit section `tech-notes`: markdown editor giống US-002 + embed detection.
- Read view: detect URL matching `*.atlassian.net` / `figma.com` / `github.com` → render preview card (icon service + URL rút gọn + domain label), clickable.
- Edit section `screenshots`: markdown editor + button "Upload ảnh" bên cạnh editor.
- Upload flow: click button → OS file picker (png/jpg/webp) → upload → auto-insert markdown `![alt](/uploads/:id)` tại vị trí con trỏ trong editor.
- Read view render `/uploads/:id` images inline (responsive, max-width container).
- `updated_by` cho `tech-notes` + `screenshots` hiển thị đúng Hùng (không nhầm với Lan).

## Scope out

- Client-side image crop/resize/compression (uploaded như-là).
- Bulk upload nhiều file cùng lúc (1 file per click v1).
- Server-side embed metadata fetch (oEmbed/OG) — chỉ client-side URL parse v1.
- Delete uploaded file (v2).
- Auth với Jira/Figma/GitHub để xem private content — embed card luôn là "link to external, external owns auth".
- Video/attachment (pdf/zip) uploads — chỉ image v1.

## Acceptance criteria (Given/When/Then)

### AC-1 — Hùng mở feature đã có của Lan
**Given** feature `dang-nhap-bang-email` có 3 business section filled bởi Lan (từ US-002)
**When** Hùng login (user `author` seeded) + mở feature edit page
**Then** thấy 5 section card; 3 business section ở read-mode hiển thị content của Lan; 2 section `tech-notes` + `screenshots` empty với button "Edit".
**And** metadata 3 section business vẫn hiện "Updated by Lan, ..." (không bị reset bởi Hùng load page).

### AC-2 — Edit tech-notes + embed card render
**Given** Hùng ở section `tech-notes` edit mode
**When** Hùng gõ: `Xem PR: https://github.com/acme/repo/pull/42` và save
**Then** API PUT section trả 200 với `updated_by = <Hùng.id>`
**And** read view render card: icon GitHub (lucide) + text "github.com/acme/repo/pull/42" + subtitle "github.com"
**And** click card → mở new tab với URL gốc.

### AC-3 — URL không whitelist render plain link
**Given** Hùng gõ `https://example.com/some-doc`
**When** save
**Then** read view render plain anchor `<a>` với URL gốc, không card.
**And** anchor có `target="_blank" rel="noopener noreferrer"`.

### AC-4 — Upload ảnh happy path
**Given** Hùng ở section `screenshots` edit mode, con trỏ ở cuối editor
**When** Hùng click "Upload ảnh" → chọn file `screenshot.png` 3 MiB
**Then** API `POST /api/v1/features/:id/uploads` trả 201 với `{ data: { id, url, sizeBytes, mimeType } }`
**And** editor auto-insert `![screenshot.png](/uploads/:id)` tại vị trí con trỏ
**And** preview pane render ảnh ngay
**And** sau save + reload, ảnh persist, hiển thị ở read view.

### AC-5 — File quá lớn
**Given** Hùng chọn file 6 MiB
**When** upload
**Then** API trả 413 `FILE_TOO_LARGE`
**And** UI toast "File quá lớn (max 5 MiB)"
**And** editor content không thay đổi (không insert markdown).

### AC-6 — MIME không hợp lệ
**Given** Hùng chọn file `.pdf` hoặc `.gif`
**When** upload
**Then** API trả 415 `UNSUPPORTED_MEDIA_TYPE`
**And** UI toast "Chỉ chấp nhận png, jpg, webp".

### AC-7 — Per-section ownership
**Given** sau khi Hùng save `tech-notes` + `screenshots`
**When** mở feature detail read view (`/projects/:slug/features/:slug`)
**Then** metadata mỗi section:
  - `business` / `user-flow` / `business-rules` → "Updated by Lan, <time>"
  - `tech-notes` / `screenshots` → "Updated by Hùng, <time>"
**And** section order vẫn fixed 5 theo thứ tự Business → User Flow → Business Rules → Tech Notes → Screenshots.

### AC-8 — Embed card render trong read view (không cần edit)
**Given** Lan đã paste URL `https://acme.atlassian.net/browse/PROJ-123` vào section `business` từ US-002
**When** Minh đọc feature detail (US-001 reader)
**Then** card Jira render với icon + URL + "atlassian.net"
**And** Minh (reader) click mở tab mới.
**And** cơ chế render card nằm trong read component, dùng chung cho mọi section có URL khớp whitelist.

### AC-9 — Unauthenticated upload
**Given** không có session
**When** POST upload endpoint
**Then** API trả 401
**And** FE block tương tự AC-8 US-002.

## UX notes (wire-level)

- **Upload widget**: button icon-only bên cạnh "Save"; loading spinner khi uploading; toast success "Uploaded ✓".
- **Embed card**: box border, padding 12px, icon 24px bên trái, 2 dòng text phải (URL + domain). Hover → border highlight, cursor pointer.
- **Tech-notes vs business markdown**: cùng editor component, chỉ khác `sectionType` prop.
- **Screenshot image render read-view**: `max-width: 100%; height: auto` responsive, `loading="lazy"`.

## Risks & open items

- **Upload size DoS**: 5 MiB × nhiều request → filesystem đầy. V1 accepted (internal, low traffic); rate limit upload endpoint (VD 10/min/user) là stretch, không blocker.
- **Path traversal / MIME spoofing**: validate MIME từ buffer magic bytes (file-type lib), không chỉ trust `Content-Type` header. Sanitize filename với uploadId UUID.
- **Embed card domain spoofing**: URL như `https://evil.com/github.com` → regex phải check hostname đúng (URL parser, không substring match).
- **Markdown image từ external URL** (không upload qua system): vẫn render read view nhưng không guarantee persist. Accepted — user chịu trách nhiệm URL external.
