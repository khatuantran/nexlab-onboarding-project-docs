## User perspective

> As an **authenticated user**, I want **to add, remove, and reorder colored skill chips on `/profile`**, so that **`SkillsCard` reflects my actual skill set instead of the 7 hardcoded fake chips ("Business Analysis", "User Story", "Figma", "SQL", "BPMN", "Product Thinking", "Onboarding Doc") that every account currently sees**.

## Scope in

- **Schema**: new table `user_skills (id UUID PK, user_id UUID FK → users.id ON DELETE CASCADE, label TEXT, color TEXT, sort_order INT, created_at TIMESTAMPTZ)` + unique index `(user_id, lower(label))` for case-insensitive dedupe.
- **Color palette**: restricted to 7 v4 accent hues — `purple | orange | green | blue | rose | amber | primary` (enum). Server rejects unknown color with 400.
- **Cap**: 12 skills per user (server-enforced 409 `SKILLS_LIMIT_EXCEEDED`; client disables "Thêm" at 12).
- **API** (2 endpoints under `/me`):
  - `GET /api/v1/me/skills` — list current skills sorted by `sort_order`. Returns `[]` when user has none.
  - `PUT /api/v1/me/skills` — **replace-all** semantics. Body `{ skills: SkillItem[] }` (0..12). Server: delete all existing + insert new in 1 transaction; assigns sequential `sort_order` from the array order; rejects intra-array duplicate labels (case-insensitive) with 400 `VALIDATION_ERROR`.
- **FE**:
  - `SkillsCard` reads `useMySkills()` — chips render with their color; empty state when no skills.
  - "Thêm" outline-dashed button on the card opens new `EditSkillsDialog`.
  - Dialog: form with up to 12 rows of `{ label input, color dropdown (7 options), up/down reorder buttons, X remove }`. "+ Thêm skill" appends a blank row (disabled at 12). Submit → PUT → invalidate `["skills","me"]` → toast "Đã cập nhật skills".

## Scope out

- Skill taxonomy / autocomplete dropdown — free text only.
- Cross-user view (teammate profile) — own profile only v1.
- Skill search / filter / endorsement.
- Drag-and-drop reorder (use up/down buttons — avoid `dnd-kit` dep).
- Per-skill description / proficiency level.

## Acceptance criteria

### AC-1 — GET shape

**Given** authenticated user **When** `GET /me/skills` **Then** 200 `{ data: SkillItem[] }`. Brand-new user → `[]`.

### AC-2 — PUT replace-all happy path

**Given** authenticated user with 0 skills
**When** `PUT /me/skills` body `{ skills: [{label:"SQL",color:"blue",sortOrder:0},{label:"Figma",color:"purple",sortOrder:1}] }`
**Then** 200 + DB has those 2 rows
**And** subsequent GET returns array of length 2 in that order.

### AC-3 — PUT cap 12

**Given** authenticated user
**When** PUT body with 13 skills
**Then** 400 `VALIDATION_ERROR` (Zod `.max(12)`) — no DB write.

### AC-4 — PUT dedupe (case-insensitive)

**Given** authenticated user
**When** PUT body has `[{label:"SQL"}, {label:"sql"}]`
**Then** 400 `VALIDATION_ERROR` mentioning duplicate label.

### AC-5 — PUT invalid color

**Given** PUT body with `color: "rainbow"` **Then** 400 `VALIDATION_ERROR`.

### AC-6 — PUT clears all

**Given** user has 3 skills **When** PUT `{ skills: [] }` **Then** 200 + DB rows for that user = 0.

### AC-7 — 401 unauth

GET + PUT both 401 `UNAUTHENTICATED` without session.

### AC-8 — Other users' rows untouched

**Given** user A has 3 skills + user B has 2 **When** A PUTs `{ skills: [] }` **Then** A's rows = 0 but B's = 2.

### AC-9 — FE renders real chips

**Given** API returns 2 skills with colors blue + purple **When** ProfilePage mounts **Then** SkillsCard shows 2 chips with matching label + accent color; the old hardcoded 7 chips are gone.

### AC-10 — FE empty state

**Given** API returns `[]` **When** SkillsCard mounts **Then** only the outline-dashed "+ Thêm" button visible — no fake skills.

### AC-11 — FE dialog flow

**Given** dialog open with 2 existing rows **When** click "+ Thêm skill" → fill label "Python" → color "green" → submit **Then** PUT sent with 3 skills (existing 2 preserving order + new one at sortOrder 2). Toast appears. Dialog closes.

### AC-12 — FE cap 12 disables Add

**Given** dialog has 12 rows **When** user looks at "+ Thêm skill" button **Then** disabled with tooltip "Tối đa 12 skill".

## UX notes

- Color dropdown shows a colored swatch + label ("Tím", "Cam", "Xanh lá", "Xanh dương", "Hồng", "Vàng", "Cam đậm").
- Reorder via up/down buttons (chevron) on each row; first row up = no-op, last row down = no-op.
- Empty row label = client-side error "Tên skill không được trống" (matches Zod `.min(1)`).
- Existing PALETTE constant from `ProfilePage.tsx` reused (move to shared SkillsCard module).
