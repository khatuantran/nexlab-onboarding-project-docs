## User perspective

### As a reader (Dev / BA)

> As a **Dev or BA**, I want **to see a catalog of all projects on the home page with one click to enter detail**, so that **I don't need to remember or type project slugs manually**.

### As an admin

> As an **Admin**, I want **to rename a project or archive it when its lifecycle ends**, so that **the catalog stays relevant and outdated projects don't confuse new onboardees**.

## Why this story now?

US-002 shipped admin-gated project creation but **không có nơi hiển thị list**. Home page `/` hiện là placeholder "Implementation in progress" — Dev login xong không biết đi đâu, phải biết slug để gõ URL direct (`/projects/demo`). Pilot sẽ tạo 5-10 project → không thể memorize. Catalog page là gap critical trước M3 launch.

Admin lifecycle (edit + archive) đi cùng vì: khi Admin dùng catalog, nhu cầu natural là "project này tên sai, đổi" hoặc "project cũ xong rồi, ẩn đi" — tách story riêng tạo ma sát không cần thiết, dồn thành 1 US lifecycle-aware.

## Scope in

- Home page `/` (authenticated) render **Project Catalog**: list projects sorted by `updated_at` desc. Mỗi row hiển thị name, description (truncate 2 lines), feature count, "updated N ago" relative time.
- Click row → navigate `/projects/:slug` (existing ProjectLandingPage US-001 T9).
- Empty state khi chưa có project nào: placeholder copy + (admin only) CTA "Tạo project đầu tiên" dẫn về CreateProjectDialog (mở ngay trigger trong header AppHeader).
- **Admin actions trên ProjectLandingPage** (`/projects/:slug`, hidden qua AdminGate):
  - Button **"Sửa project"** → mở `EditProjectDialog` (Radix Dialog, form name + description, slug read-only).
  - Button **"Lưu trữ"** → native `window.confirm("Lưu trữ project? Sẽ ẩn khỏi catalog.")` → PATCH archive → redirect về `/` (catalog sẽ không còn project này).
- Schema: thêm `projects.archived_at timestamptz nullable` (Drizzle migration).
- API:
  - `GET /api/v1/projects` → list non-archived, sorted updated desc.
  - `PATCH /api/v1/projects/:slug` → admin update name + description. Slug immutable (ignore nếu có trong body).
  - `POST /api/v1/projects/:slug/archive` → admin set `archived_at = NOW()`. 204 response.

## Scope out

- Unarchive / recover archived project → v2.
- Hard delete (CASCADE sections/features) → v2.
- Bulk operations (archive multiple) → v2.
- Filter catalog by owner / date / search → v2 (header search đã cover content search; catalog-specific filter defer).
- Project membership / per-project permission → out (FR-PROJ-001 access model: mọi authenticated user read-all).
- Rename slug → explicitly out (FR-PROJ-002 slug immutable).
- Audit log / activity feed trên catalog row → defer US-005+.
- Project cover image / icon → defer.

## Acceptance criteria

### AC-1 — Authenticated user thấy catalog ở `/`

**Given** Dev hoặc Admin login thành công
**When** browser landed ở `/` (home)
**Then** hiển thị heading "Onboarding Portal" + grid các project card
**And** mỗi card hiện name, description (truncate 2 dòng), "N feature" count, relative time "cập nhật 2 giờ trước"
**And** sort theo `updated_at` desc (project mới đổi nhất lên đầu).

### AC-2 — Click card navigate detail

**Given** Dev ở `/` có catalog render
**When** Dev click project card (hoặc title link)
**Then** browser navigate `/projects/:slug` → ProjectLandingPage (US-001 AC-3 render).

### AC-3 — Catalog empty state

**Given** chưa có project nào trong DB (hoặc mọi project archived)
**When** user landed `/`
**Then** hiển thị empty placeholder "Chưa có project nào trong catalog"
**And** nếu user là admin → kèm button "Tạo project đầu tiên" click mở CreateProjectDialog
**And** nếu user là author → chỉ placeholder, không CTA.

### AC-4 — Catalog loại archived project

**Given** có 3 project, 1 trong đó có `archived_at` set
**When** user load `/`
**Then** chỉ hiển thị 2 project non-archived
**And** direct URL `/projects/:slug` của archived project return 404 `PROJECT_NOT_FOUND`.

### AC-5 — Admin edit project metadata

**Given** Admin ở `/projects/pilot-project`
**When** Admin click "Sửa project", đổi name thành "Pilot Project V2", description, click "Lưu"
**Then** API `PATCH /api/v1/projects/pilot-project` trả 200 với project cập nhật
**And** dialog close, ProjectLandingPage heading refresh tên mới
**And** sonner toast "Đã cập nhật project"
**And** reload trang → tên mới persist.

### AC-6 — Slug không edit được

**Given** Admin mở EditProjectDialog
**When** Admin inspect form
**Then** field slug không xuất hiện (hoặc readonly với hint "Slug không đổi được sau khi tạo")
**And** PATCH body có field `slug` → server ignore (không error, không update).

### AC-7 — Admin archive project

**Given** Admin ở `/projects/pilot-project`
**When** Admin click "Lưu trữ" → confirm dialog → click OK
**Then** API `POST /api/v1/projects/pilot-project/archive` trả 204
**And** browser redirect về `/` (catalog)
**And** catalog **không** hiển thị "Pilot Project" nữa
**And** sonner toast "Đã lưu trữ project".

### AC-8 — Non-admin không thấy admin actions

**Given** Dev (role author) ở `/projects/pilot-project`
**When** Dev inspect page
**Then** không có button "Sửa project" hoặc "Lưu trữ"
**And** trực tiếp gọi `PATCH /api/v1/projects/pilot-project` → 403 `FORBIDDEN`
**And** trực tiếp gọi `POST /api/v1/projects/pilot-project/archive` → 403 `FORBIDDEN`.

### AC-9 — Archived project 404 on direct access

**Given** project "old-pilot" đã archived
**When** user (bất kỳ role) gõ `/projects/old-pilot`
**Then** hiển thị "Không tìm thấy project 'old-pilot'" (reuse US-001 error page)
**And** API `GET /api/v1/projects/old-pilot` trả 404 `PROJECT_NOT_FOUND`.

## Risks & open items

- **Archive semantics ambiguity**: "Archive rồi unarchive sau" cần row vẫn tồn tại với `archived_at IS NULL` restore. V1 scope chỉ archive 1 chiều. Document rõ trong FR-PROJ-002 rationale.
- **Search scope after archive**: FTS currently full-table scan; cần filter out archived projects. Test case: search keyword match archived feature → 0 results.
- **Slug immutable decision**: nếu team muốn rename slug sau → phải ship redirect map trong v2. Track as risk.
- **Empty catalog vs no-access**: cả 2 cùng hiển thị empty placeholder. V1 không phân biệt vì không có membership table.
- **UI spec gate**: `home.md` + `edit-project-dialog.md` chưa tồn tại — Draft story, cần user approve Gate 1 của 2 UI spec này trước khi flip Ready.
