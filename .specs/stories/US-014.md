## User perspective

> As an **authenticated user on `/`**, I want **the hero tiles + "Đủ doc" filter chip to reflect real workspace numbers**, so that **the first thing I see is truthful instead of `Contributors=8` / `Onboard TB=2.3h` / `featureCount > 0` proxy**.

## Scope in

- New endpoint `GET /api/v1/workspace/stats` (auth required) → `{ projectCount, featuresDocumented, contributorsActive }`.
  - `projectCount` = `COUNT(projects)` non-archived.
  - `featuresDocumented` = `COUNT(features)` non-archived where ALL 5 sections have `length(body) > 0`.
  - `contributorsActive` = `COUNT(DISTINCT sections.updated_by)` where `updated_at >= NOW() - INTERVAL '30 days'`.
- Extend `ProjectSummary` (shared) with `filledSectionCount: number` — sum of non-empty sections across this project's non-archived features.
- FE: HomePage hero goes 4-tile → **3-tile** (Projects / Đủ tài liệu / Đóng góp 30d). "Onboard TB" tile dropped per 2026-05-16 triage.
- FE: `filter === "doc"` chip now `(p) => p.filledSectionCount === p.featureCount * 5 && p.featureCount > 0` (replaces `p.featureCount > 0` placeholder).

## Scope out

- `onboardingMedianHours` tile — dropped, no DB signal.
- Time-series / sparkline / week-over-week.
- Per-project sparkline.
- Caching layer (compute on-demand v1; revisit when N > 100 projects).

## Acceptance criteria

### AC-1 — endpoint shape

**Given** authenticated user **When** `GET /workspace/stats` **Then** 200 `{ data: { projectCount: number, featuresDocumented: number, contributorsActive: number } }`.

### AC-2 — featuresDocumented requires 5/5

**Given** project P has 2 features, F1 with all 5 sections filled, F2 with 4 of 5 **When** GET stats **Then** `featuresDocumented` counts F1 (1), not F2.

### AC-3 — contributorsActive 30d window

**Given** user A edited a section 5d ago, user B 60d ago **When** GET stats **Then** `contributorsActive === 1` (only A).

### AC-4 — archived excluded

**Given** archived project / feature exists **Then** they are not counted in any of the 3 numbers.

### AC-5 — unauth 401

**Given** no session **When** GET /workspace/stats **Then** 401 `UNAUTHENTICATED`.

### AC-6 — ProjectSummary.filledSectionCount

**Given** project P has 2 features × 5 sections each with 7 non-empty bodies total **When** `GET /projects` **Then** the P row carries `filledSectionCount: 7`.

### AC-7 — HomePage hero renders 3 tiles

**Given** stats endpoint returns `{ projectCount: 5, featuresDocumented: 12, contributorsActive: 3 }` **When** HomePage mounts **Then** 3 tiles render with those values; no 4th "Onboard TB" tile.

### AC-8 — "Đủ doc" filter

**Given** project A featureCount=2 filledSectionCount=10 (= 2×5) AND project B featureCount=2 filledSectionCount=8 **When** filter chip "Đủ doc" active **Then** only A visible.

### AC-9 — loading state

**Given** stats request in flight **When** HomePage mounts **Then** tile values render as skeleton `—` placeholder (current behavior preserved).
