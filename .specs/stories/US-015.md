## User perspective

> As an **authenticated user on `/profile`**, I want **the 4 stat tiles to reflect my real contribution numbers**, so that **`4 / 18 / 42 / 12` hardcoded stops misrepresenting me**.

## Scope in

- New endpoint `GET /api/v1/me/stats` (auth) → `{ projectsTouched, featuresDocumented, totalEdits, sectionsCompleted }`.
  - `projectsTouched` = `COUNT(DISTINCT features.project_id)` for sections where `updated_by = me`.
  - `featuresDocumented` = `COUNT(DISTINCT sections.feature_id)` where `updated_by = me` AND that feature has all 5 sections non-empty (any author).
  - `totalEdits` = `COUNT(sections)` where `updated_by = me` (current rows; no history).
  - `sectionsCompleted` = `COUNT(sections)` where `updated_by = me` AND `length(body) > 0`.
- FE: `ProfilePage` `StatsCard` reads `useMeStats()`; skeleton during load.

## Scope out

- Historical edit count (PUT overwrites).
- Trend / streak.
- Admin viewing another user's stats.

## Acceptance criteria

### AC-1 — shape

`GET /me/stats` → 200 `{ data: { projectsTouched, featuresDocumented, totalEdits, sectionsCompleted } }`.

### AC-2 — brand-new user

**Given** user never edited a section **When** GET /me/stats **Then** all 4 counts = 0.

### AC-3 — projectsTouched

**Given** user has edited sections in 3 distinct projects **Then** `projectsTouched === 3`.

### AC-4 — archived feature excluded

**Given** user edited a section in an archived feature **Then** that doesn't increment any count.

### AC-5 — sectionsCompleted vs totalEdits

**Given** user has 5 section rows where `updated_by = me`, 3 with non-empty body **Then** `totalEdits === 5`, `sectionsCompleted === 3`.

### AC-6 — 401 unauth

No session → 401.

### AC-7 — FE StatsCard renders real counts

**Given** stats endpoint returns `{2, 6, 14, 4}` **When** ProfilePage mounts **Then** 4 tiles show those values; not `4 / 18 / 42 / 12`.
