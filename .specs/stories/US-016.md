## User perspective

> As an **authenticated user**, I want **`/profile` "Projects tham gia" card to list projects I touched, ranked by my latest edit**, so that **the 4 hardcoded entries (USthree / E2E Backend / Catalog Search / Momo Payment) get replaced with real work**.

## Scope in

- New endpoint `GET /api/v1/me/recent-projects?limit=4` (auth) → `[{ slug, name, lastTouchedAt, sectionsTouched }]`.
  - Source: `sections.updated_by = me`. Group by `project_id`, order by `max(sections.updated_at) DESC`, limit (default 4, max 20).
  - `sectionsTouched` = count of section rows for that project where `updated_by = me`.
  - Exclude rows from archived projects + archived features.
- FE: `ProfilePage` `RecentProjectsCard` reads `useMyRecentProjects(4)`. Each row links to `/projects/:slug`. Render `sectionsTouched`-based progress bar `pct = sectionsTouched / (featureCount * 5)` when known — but featureCount isn't in this projection. Compromise: drop pct entirely, replace with last-edit relative time (cleaner truth).

## Scope out

- Per-project filled% across all authors (out — that's `filledSectionCount`, separate).
- Pinning / favorites.

## Acceptance criteria

### AC-1 — endpoint shape + ranking

**Given** user edited project A 1d ago, B 5d ago, C 30d ago **When** GET /me/recent-projects?limit=4 **Then** array `[A, B, C]` in that order.

### AC-2 — limit cap

`?limit=10` accepted; `?limit=100` clamped to 20; default 4 when missing.

### AC-3 — empty

**Given** brand-new user **Then** `data: []`. FE renders empty state "Chưa có dự án nào — bắt đầu chỉnh sửa để xuất hiện ở đây."

### AC-4 — archive excluded

**Given** user edited section in archived project / archived feature **Then** that project NOT in response.

### AC-5 — FE renders rows + empty

ProfilePage RecentProjectsCard list reflects API response; row labels show project name + relative time (`formatRelativeVi`) + `sectionsTouched` badge.
