## User perspective

> As a **new developer (Minh)**, I want to **log in, browse projects, search features by keyword, and read a feature's 5-section detail**, so that I can **onboard to a new project without 1-1 mentoring từ senior**.

## Why this story first?

US-001 là **reader happy-path** — justify toàn bộ upstream work (DB, schema, API read, FE shell, auth). Chạy được US-001 end-to-end = MVP có value thực sự dù chưa có authoring UI (seed data là đủ để demo).

---

## Scope in

- Login page `/login` + session cookie `sid` + logout button trong header.
- Project landing `/projects/:slug` — list feature sắp xếp updated-desc, hiển thị section-filled indicator ("N/5 filled").
- Feature detail `/projects/:slug/features/:slug` — render 5 section theo thứ tự cố định, mỗi section có tiêu đề rõ ràng. Empty section render placeholder, không hidden.
- Search bar trong header — query ≥ 1 ký tự → top-20 feature + snippet highlight + rank. Scope search: cross-project (không filter theo project v1).
- Auth guard: route `/projects/*` yêu cầu session; unauthenticated → redirect `/login?next=<originalUrl>`.

## Scope out

- Mọi write/edit flow (create/update project/feature/section) → US-002 + US-003.
- Embed preview card (Jira/Figma/GitHub URL render card) → US-003 (dù reader sẽ thấy sau khi Hùng paste, nhưng FE code cho embed nằm ở US-003).
- Upload screenshot → US-003.
- User invite / admin UI → v2.
- Password reset → v2.
- Remember-me / persistent session → sau pilot.

## Acceptance criteria (Given/When/Then)

### AC-1 — Login happy path
**Given** Minh có tài khoản hợp lệ được admin tạo sẵn (seed hoặc admin invite)
**When** Minh nhập email + password đúng ở `/login` và submit
**Then** hệ thống set cookie `sid` httpOnly và redirect về `/` (hoặc `next` param nếu có)
**And** header hiển thị tên của Minh + nút Logout.

### AC-2 — Login invalid credentials
**Given** Minh ở `/login`
**When** Minh nhập email không tồn tại, hoặc email đúng nhưng sai password
**Then** hệ thống trả HTTP 401 `INVALID_CREDENTIALS`
**And** UI hiển thị inline error "Email hoặc mật khẩu không đúng" — không phân biệt 2 case (chống user enumeration).

### AC-3 — Project landing
**Given** Minh đã login
**When** Minh truy cập `/projects/demo` (seeded project)
**Then** trang hiển thị tên + mô tả project
**And** list các feature sắp xếp theo `updated_at` giảm dần
**And** mỗi feature row hiển thị: title, slug, "updated 2h ago" (relative), indicator "3/5 sections filled".
**And** response server-side < 300ms p95 (≤ 500 feature).

### AC-4 — Project empty state
**Given** project tồn tại nhưng chưa có feature nào
**When** Minh mở landing
**Then** trang hiển thị placeholder "Chưa có feature nào trong project này" — không crash, không 404.

### AC-5 — Feature detail render 5 section
**Given** feature `login-with-email` có seed đầy đủ 5 section
**When** Minh click vào feature từ landing
**Then** URL đổi thành `/projects/demo/features/login-with-email`
**And** trang render đúng 5 section theo thứ tự: Business → User Flow → Business Rules → Tech Notes → Screenshots
**And** mỗi section có heading + body markdown đã render sang HTML an toàn (sanitize).

### AC-6 — Feature detail empty section
**Given** feature có section `tech-notes` còn trống (body rỗng)
**When** Minh xem feature detail
**Then** section `tech-notes` vẫn render với heading + placeholder "Chưa có nội dung" — không hidden.
**And** các section khác vẫn render bình thường.

### AC-7 — Search happy path
**Given** Minh đã login + có ít nhất 1 feature match keyword "login"
**When** Minh gõ "login" vào search bar header và submit (Enter hoặc click icon)
**Then** trang `/search?q=login` hiển thị top-20 feature khớp
**And** mỗi kết quả hiển thị: title, project slug, snippet 160-200 chars có highlight `<mark>` keyword, rank score ẩn (dùng để sort)
**And** kết quả sắp xếp theo `ts_rank` desc.
**And** response < 500ms p95.

### AC-8 — Search empty query
**Given** Minh ở bất kỳ page
**When** Minh submit search form với query rỗng hoặc chỉ khoảng trắng
**Then** UI block submit (disable button hoặc toast "Nhập từ khoá để tìm")
**And** không gọi API (client-side guard).

### AC-9 — Search no result
**Given** Minh search keyword không khớp feature nào (VD "zzzzz")
**When** request trả về
**Then** trang hiển thị "Không tìm thấy feature nào khớp 'zzzzz'" + link "Về trang chủ"
**And** HTTP status vẫn 200 (không 404 — no-result không phải error).

### AC-10 — Unauthenticated redirect
**Given** Minh chưa login (không có cookie `sid` hoặc session expired)
**When** Minh truy cập trực tiếp `/projects/demo/features/login-with-email`
**Then** browser redirect `/login?next=%2Fprojects%2Fdemo%2Ffeatures%2Flogin-with-email`
**And** sau khi login thành công, redirect về URL gốc.

### AC-11 — Logout
**Given** Minh đã login
**When** Minh click Logout ở header
**Then** session bị destroy ở Redis, cookie `sid` clear
**And** browser redirect `/login`
**And** truy cập `/projects/*` sau đó sẽ redirect lại login (AC-10).

## UX notes (wire-level, không có Figma v1)

- **Layout**: single-column, max-width ~900px cho feature detail; 2-col landing (project meta sidebar trái, feature list phải). Mobile responsive nhưng không prioritize.
- **Header**: logo trái + search input giữa (desktop) + avatar/Logout phải.
- **Feature detail section**: mỗi section là card có heading + content; section order fix không để user collapse v1.
- **Section indicator**: chấm xanh "●" nếu filled, chấm xám "○" nếu empty, render 5 chấm cạnh title. "3/5" số ở cạnh.
- **Search result snippet**: render dưới dạng `<p>`, highlight `<mark class="bg-yellow-200">` (via `ts_headline` markup server-side + sanitize FE).

## Risks & open items

- **Markdown sanitize**: dùng `markdown-it` + `DOMPurify` (FE) hay server render? → Sẽ chốt trong task T7. Default: server render markdown → HTML rồi FE inject qua `dangerouslySetInnerHTML` sau DOMPurify sanitize.
- **Search scope v1 cross-project hay per-project?** → Chọn cross-project cho MVP (đơn giản hơn, value cao hơn cho new dev chưa biết project nào). Filter project sẽ là task phụ nếu còn thời gian.
- **Session store**: Redis down → login fail. Acceptable v1 (Redis là hard dep đã document trong ADR-001). Fallback JWT chưa implement.
