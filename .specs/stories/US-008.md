## User perspective

> As an **Admin**, I want **to archive a feature inside a project when its scope is dropped or merged into another feature**, so that **the project's feature catalog only shows what's still relevant without losing the historical sections + uploads**.

## Why this story now?

US-004 đã ship archive cho project nhưng không có equivalent cho feature. Khi admin cần dọn catalog feature inside 1 project — VD pilot merge 2 feature, hoặc feature spike bị abandon — cách duy nhất hiện tại là rename thành "Deprecated — XYZ" hoặc xóa cả project. Cả 2 đều sai. Feature archive là gap nhỏ, blocker thấp, nhưng feedback từ pilot 2026-05+ sẽ cần ngay. Ship bây giờ trước M3 launch.

---

## Scope in

- Schema: thêm `features.archived_at timestamptz nullable` qua hand-written migration.
- API:
  - `POST /api/v1/projects/:slug/features/:featureSlug/archive` → admin set `archived_at = NOW()`. 204 response, idempotent.
  - `GET /api/v1/projects/:slug` response `features[]` filter `WHERE features.archived_at IS NULL`.
  - `GET /api/v1/projects/:slug/features/:featureSlug` filter archived → 404 `FEATURE_NOT_FOUND`.
- FE:
  - `FeatureActionsMenu` component (⋯ overflow) trên `FeatureCard`, admin-only.
  - Single menu item "Lưu trữ feature" — native `window.confirm` → POST → toast + invalidate cache.
  - Hide `ChevronRight` khi admin (BUG-004 lesson — không cho 2 affordance đè lên nhau).
- Reuse pattern: `ProjectActionsMenu` (US-004), `useArchiveProject` mutation hook.

## Scope out

- Unarchive / restore feature → defer v2.
- Hard delete + cascade sections/uploads → defer v2 (user xác nhận "chỉ soft delete").
- Edit feature dialog (rename title) — defer; menu chỉ có 1 item "Lưu trữ" lần này.
- Bulk archive nhiều feature → defer.
- Filter archived feature ra khỏi search v2 → defer; archived feature search-match → click → 404 page, known limitation Phase 1.
- Audit log archive events.
- Author (non-admin) archive — explicit out vì destructive (consistency với project archive admin-only).

## Acceptance criteria

### AC-1 — Admin thấy ⋯ overflow trên FeatureCard

**Given** Admin login + ở `/projects/pilot`
**When** Admin nhìn 1 FeatureCard
**Then** card hiển thị ⋯ button overflow ở top-right corner
**And** ChevronRight (affordance navigation) bị ẩn đi để tránh overlap (per BUG-004).

### AC-2 — Click ⋯ mở menu với "Lưu trữ feature"

**Given** Admin ở FeatureCard có ⋯ visible
**When** Admin click ⋯
**Then** dropdown menu hiển thị 1 item "Lưu trữ feature" với Archive icon
**And** click vào item không navigate (event stopPropagation tránh trigger Link wrap parent).

### AC-3 — Confirm dialog trước archive

**Given** Admin click "Lưu trữ feature"
**When** native `window.confirm` xuất hiện
**Then** copy hỏi "Lưu trữ feature '<title>'? Feature sẽ ẩn khỏi catalog, sections + uploads giữ nguyên."
**And** click Cancel → không call API, menu close.

### AC-4 — Archive happy path

**Given** Admin click OK trong confirm
**When** FE POST `/api/v1/projects/pilot/features/dang-nhap/archive` với admin cookie
**Then** server trả 204 No Content
**And** FE invalidate `projectKeys.byId("pilot")` + sonner toast "Đã lưu trữ feature"
**And** FeatureCard biến mất khỏi list (refetched `GET /projects/pilot` không return feature đó nữa).

### AC-5 — Archived feature 404 trên direct URL

**Given** feature "dang-nhap" đã được archived
**When** user (bất kỳ role) gõ `/projects/pilot/features/dang-nhap`
**Then** API `GET /projects/pilot/features/dang-nhap` trả 404 `FEATURE_NOT_FOUND`
**And** FeatureDetailPage hiển thị "Không tìm thấy feature 'dang-nhap'" (reuse US-001 error UI).

### AC-6 — Idempotent archive

**Given** feature "dang-nhap" đã archived (archived_at set)
**When** Admin gọi lại `POST .../archive`
**Then** server vẫn trả 204 (no-op, archived_at giữ giá trị cũ).

### AC-7 — Non-admin bị 403

**Given** Author (role !== admin) gọi `POST /api/v1/projects/pilot/features/dang-nhap/archive`
**When** request đến server
**Then** server trả 403 `FORBIDDEN`
**And** UI: Author không thấy ⋯ menu (admin gate) — chỉ thấy ChevronRight như cũ.

### AC-8 — Unauthenticated bị 401

**Given** user chưa login
**When** call `POST .../archive`
**Then** server trả 401 (handled bởi `requireAuth` middleware).

## UX notes (wire-level)

- **FeatureCard** overlay: `absolute right-2 top-2 z-10` cho `FeatureActionsMenu`. Wrap stopPropagation trên Click + KeyDown để Radix dropdown trigger không bubble lên Link wrap card.
- **FeatureActionsMenu**: clone visual của `ProjectActionsMenu` — `MoreHorizontal` icon trigger (Button variant ghost size icon), DropdownContent align end, 1 item destructive-tone "Lưu trữ feature" (Archive icon + `text-destructive focus:bg-destructive/10`).
- **Confirm UX**: native `window.confirm` (match pattern project archive — không tạo AlertDialog mới).
- **FeatureCard chevron**: ẩn khi admin (giống ProjectCard post-BUG-004). Author vẫn thấy chevron để có affordance click-navigate.
- **Toast**: `sonner` success "Đã lưu trữ feature"; error 403 → "Bạn không có quyền", 404 → "Feature không tồn tại".
- Reference UI spec update: `.specs/ui/project-landing.md` thêm row Interactions "Click ⋯ trên FeatureCard (admin)"; `.specs/ui/design-system.md` §5.3 thêm row `FeatureActionsMenu`.

## Risks & open items

- **Search shows archived feature**: known limitation — Phase 1 không filter. Mitigation: click qua đã 404 nên user discover ngay. Track follow-up.
- **Author UX no-op**: Author không thấy menu nên không biết archive khả dụng — intentional (admin-only). Documentation trong CONTRIBUTING không cần update vì internal.
- **Cascade safety**: sections.feature_id + uploads.feature_id đều có ON DELETE CASCADE — archive không trigger delete (chỉ UPDATE archived_at). Test cover.
- **FeatureCard `ChevronRight` hidden cho admin** có thể làm admin tưởng card không click được. Mitigation: card-wrap vẫn là `<Link>` hover-elevate, vẫn navigate khi click vùng ngoài menu trigger. Same UX với ProjectCard hiện tại — chấp nhận parity.
