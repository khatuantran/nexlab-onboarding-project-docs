# BUG-002 — Theme toggle cần 2 click để chuyển từ Dark về Light

<!-- template: 05-bug-template.md@0.1 -->

## Metadata

- **Severity**: 🟡 Medium
- **Priority**: P1
- **Status**: Open
- **Reported by**: @khatrannexlab
- **Reported at**: 2026-04-25
- **Fixed at commit**: `<hash>`
- **Environment**: dev (and any env where OS prefers-color-scheme matches the dark/light state)

## Reproduction

1. Mở app trên máy có OS `prefers-color-scheme: dark` (hoặc `localStorage.theme = "dark"` trực tiếp).
2. App ở dark mode (icon Moon trong AppHeader).
3. Click Theme toggle button.
4. Quan sát: icon đổi từ Moon → Monitor (system) nhưng page vẫn dark.
5. Click lần nữa → page mới về light (icon Sun).

**Prerequisites**: OS prefer-color-scheme = dark, hoặc app vừa restore `theme=dark` từ localStorage.

**Frequency**: always khi OS resolved = dark.

## Expected vs Actual

- **Expected**: 1 click trên toggle button khi đang ở dark phải trả về light ngay (visual feedback đồng bộ với click).
- **Actual**: Click 1 chuyển state `dark → system`; vì OS prefers-color-scheme = dark nên `resolvedTheme` vẫn = dark → page không đổi, chỉ icon đổi. Cần click 2 mới rotate sang `light`.
- **Spec reference**: [.specs/ui/design-system.md §1.2 — Theme cycling](../ui/design-system.md). Spec mô tả 3-state cycle nhưng không đảm bảo "1 click = 1 visual change", lộ ra UX surprise.

## Scope

- **Affected files**:
  - [apps/web/src/lib/theme.tsx](../../apps/web/src/lib/theme.tsx) — `cycleTheme` rotates `system → light → dark → system`.
  - [apps/web/src/components/layout/ThemeToggle.tsx](../../apps/web/src/components/layout/ThemeToggle.tsx) — invokes `cycleTheme`.
  - [apps/web/tests/components/ThemeToggle.test.tsx](../../apps/web/tests/components/ThemeToggle.test.tsx) — encodes the buggy 3-state assertion ("cycles light → dark → system on each click").
- **Affected screens**: AppHeader on every authenticated route + LoginPage header (anywhere ThemeToggle renders).

## Root cause

`cycleTheme` rotates through 3 themes (`system → light → dark → system`). Icon binds to `theme` (the explicit choice), not `resolvedTheme` (what the user sees). When `theme = dark` and the OS prefers dark:

- Click → `theme = system`. Icon: Moon → Monitor.
- `resolve("system")` → reads `matchMedia("(prefers-color-scheme: dark)")` → `dark`. Page stays dark.
- The user perceives "I clicked, icon moved, but the page didn't change" → assumes the toggle is broken / requires another click.

## Fix approach

Drop `system` from the click cycle; toggle becomes a 2-state switch driven by **resolvedTheme** so that every click flips the visible mode:

- If current resolved is dark → set `theme = "light"`.
- If current resolved is light → set `theme = "dark"`.

`system` remains the default when no theme is stored (OS detection still works on first load + when localStorage is cleared). Explicit "system" selection via this button is removed; if needed in the future it can be re-added behind a long-press / dropdown menu (out of scope for this fix).

Icon binding moves to `resolvedTheme` so Sun = currently-showing-light and Moon = currently-showing-dark, matching what the user sees.

## Regression test

**File**: [apps/web/tests/components/ThemeToggle.test.tsx](../../apps/web/tests/components/ThemeToggle.test.tsx) (new test block "BUG-002 dark → click → light single hop").

**Assertion**:

1. Mock `matchMedia` to return `prefers-color-scheme: dark`.
2. Set `localStorage.theme = "dark"` and mount provider.
3. Confirm initial state: `theme=dark, resolvedTheme=dark, document.documentElement.classList.contains("dark") === true`.
4. Click the toggle once.
5. Expect: `theme="light", resolvedTheme="light", document.documentElement.classList.contains("dark") === false` after a single click.

The existing "cycles light → dark → system on each click" assertion is rewritten to the new 2-state contract ("cycles light ↔ dark on each click").
